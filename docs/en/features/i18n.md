# Internationalization

Bunshelf has built-in internationalization (i18n) support, making it easy to create documentation in multiple languages.

## Configuration

### Enable Languages

Add languages in `docs/config.yaml`:

```yaml
defaultLocale: en
locales:
  en:
    indexPage: /intro
    localePrefix: english
  tr:
    indexPage: /intro
    localePrefix: turkish
  de:
    indexPage: /intro
    localePrefix: german
```

### Locale Options

Each locale can have:

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `indexPage` | string | "/intro" | Landing page for this locale |
| `localePrefix` | string | locale code | URL prefix (e.g., `turkish` instead of `tr`) |

### Locale Format

Use ISO 639-1 language codes:

| Code | Language |
|------|----------|
| `en` | English |
| `tr` | Turkish |
| `de` | German |
| `fr` | French |
| `es` | Spanish |
| `ja` | Japanese |
| `zh` | Chinese |
| `ko` | Korean |

For regional variants, use ISO 3166-1:

| Code | Language |
|------|----------|
| `en-us` | English (US) |
| `en-gb` | English (UK) |
| `pt-br` | Portuguese (Brazil) |
| `zh-cn` | Chinese (Simplified) |

## Directory Structure

Organize content by locale:

```
docs/
├── config.yaml
├── en/
│   ├── intro.md
│   ├── getting-started/
│   │   ├── installation.md
│   │   └── quick-start.md
│   └── advanced/
│       └── configuration.md
├── tr/
│   ├── intro.md
│   ├── getting-started/
│   │   ├── installation.md
│   │   └── quick-start.md
│   └── advanced/
│       └── configuration.md
└── de/
    └── intro.md
```

## URL Structure

### Default Locale (en)

English content is served at the root (no prefix):

- `/intro` → `docs/en/intro.md`
- `/getting-started/installation` → `docs/en/getting-started/installation.md`

### Other Locales

Other languages use the `localePrefix` in the URL:

- `/turkish/intro` → `docs/tr/intro.md`
- `/german/intro` → `docs/de/intro.md`

::: tip Note
The `localePrefix` can be customized. For example, using `turkish` instead of `tr` makes URLs more readable.
:::

### Adding a New Language

1. Create the translation file:

```bash
touch src/i18n/translations/de.json
```

2. Add translations:


## Locale Switcher

Users can switch languages using the locale switcher in the sidebar:

- Shows all available locales
- Preserves the current page path
- Redirects to the same page in the selected language

## Sidebar per Locale

Configure different sidebars for each language. Note that you don't need to add the locale prefix to `href` values - it's added automatically:

```yaml
sidebar:
  en:
    - label: Getting Started
      items:
        - label: Introduction
          href: /intro
        - label: Installation
          href: /getting-started/installation
  tr:
    - label: Başlarken
      items:
        - label: Giriş
          href: /intro
        - label: Kurulum
          href: /getting-started/installation
```

::: tip Automatic Prefix
For non-default locales, the `localePrefix` is automatically added to sidebar hrefs. So `/intro` becomes `/turkish/intro` for Turkish locale.
:::

## Fallback Behavior

### Missing Content

If a page doesn't exist in a locale:

- {red}404 page{/} is shown in the selected locale
- User can switch to another language

### Missing Translation

If a translation key is missing:

- Falls back to English translation
- Falls back to the key name itself

## Best Practices

### Content Organization

- Keep the same structure across all locales
- Translate all pages, not just some
- Update all languages when adding new content

### Translation Quality

- Use native speakers for translations
- Keep translations consistent with your brand voice
- Review translations for technical accuracy

### Maintenance

```yaml
# Good practice: Add order to organize sidebar
---
title: Installation
order: 1
---
```

::: tip Pro Tip
Use the same `order` values across locales to maintain consistent sidebar ordering.
:::

## Example: Full Multilingual Setup

```yaml
# docs/config.yaml
title: My Documentation
description: Documentation in multiple languages
defaultLocale: en
locales:
  en:
    indexPage: /intro
    localePrefix: english
  tr:
    indexPage: /intro
    localePrefix: turkish
  de:
    indexPage: /intro
    localePrefix: german
theme:
  default: light
logo: /assets/images/logo.webp
sidebar:
  en:
    - label: Getting Started
      items:
        - label: Introduction
          href: /intro
        - label: Installation
          href: /getting-started/installation
  tr:
    - label: Başlarken
      items:
        - label: Giriş
          href: /intro
        - label: Kurulum
          href: /getting-started/installation
  de:
    - label: Erste Schritte
      items:
        - label: Einführung
          href: /intro
        - label: Installation
          href: /getting-started/installation
```
